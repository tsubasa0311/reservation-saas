# 画面遷移図・画面仕様

## ルーティング全体像

```
/                           ← ログインページへリダイレクト
/login                      ← 共通ログイン（店舗ID + ログインID + パスワード）
/logout

[プレイヤー]
/player                     ← 自分の予約一覧（仮/確定/キャンセル別タブ）
/player/new                 ← 新規予約入力
/player/reservations/[id]   ← 予約詳細（自分のみ）
/player/sales               ← 自分の売上・バック額
/player/password            ← パスワード変更

[オーナー]
/owner                      ← ダッシュボード
/owner/reservations         ← 予約一覧（ステータス・期間・プレイヤーでフィルタ）
/owner/reservations/[id]    ← 予約詳細・確定・キャンセル・編集
/owner/players              ← プレイヤー一覧
/owner/players/new          ← プレイヤー追加
/owner/players/[id]         ← プレイヤー編集（上書き料金・バック率含む）
/owner/courses              ← コースマスタ編集
/owner/nomination           ← 指名料マスタ編集
/owner/extensions           ← 延長料金マスタ編集
/owner/options              ← オプションマスタ編集
/owner/settings             ← 店舗設定・交通費設定
/owner/password             ← パスワード変更

[スーパー管理者（つばさ）]
/admin                      ← 全店舗ダッシュボード
/admin/stores               ← 店舗一覧
/admin/stores/new           ← 店舗追加
/admin/stores/[id]          ← 店舗詳細・オーナー発行・凍結
/admin/stores/[id]/impersonate  ← なりすましログイン
```

## プレイヤー画面詳細

### /player（予約一覧）

**表示内容**
- 上部：自分の名前 / 今月の売上 / 今月のバック額
- タブ：仮予約 / 確定済 / キャンセル / 全て
- リスト：開始日時 / 顧客名 / コース / 合計金額 / ステータス
- 新規予約ボタン（画面右下固定、スマホ優先）

### /player/new（新規予約入力）

入力項目（順番重要・スマホで上から順に入力しやすく）

1. **顧客区分**：ラジオボタン `新規 / 会員`
2. **顧客名**：テキスト入力
3. **予約方法**：ラジオ `LINE / メール / DM / 電話`
4. **合流方法**：ラジオ `待ち合わせ / ホテル先入り / 自宅 / DMにて`
5. **開始日時**：日付ピッカー + 時分
6. **指名種別**：ラジオ `初指名 / 本指名`（→ 指名料自動表示）
7. **コース**：セレクト（自分に該当する有効なコースのみ / 料金表示）
8. **延長**：セレクト（なし + 延長マスタ全件）
9. **オプション**：チェックボックス複数選択 + 個数
10. **交通費**：数値入力（デフォルト0）
11. **支払い方法**：ラジオ `現金 / クレジットカード`

**画面下部**
- 合計金額プレビュー（リアルタイム計算）
- 自分のバック額プレビュー
- 送信ボタン（送信後は仮予約として保存）

**バリデーション**
- 必須項目の未入力チェック
- 開始日時は現在以降（過去日時は警告表示で送信は許可、入力ミス対策）

### /player/reservations/[id]（予約詳細）

- 予約内容全て表示
- 仮予約状態なら「編集」「取り消し」可能
- 確定後は閲覧のみ

### /player/sales（売上）

- 期間セレクタ：今日 / 今週 / 今月 / 先月 / カスタム
- サマリ：売上総額 / バック額 / 件数
- 明細：予約単位のリスト
- CSVダウンロード

## オーナー画面詳細

### /owner（ダッシュボード）

- 期間セレクタ（今日/今週/今月/先月/カスタム）
- 店舗全体サマリ：売上 / バック総額 / 店舗取り分 / 件数
- プレイヤー別売上ランキング（棒グラフ）
- 仮予約の未確定件数（クリックで予約一覧へ）

### /owner/reservations（予約一覧）

**フィルタ**
- ステータス（仮/確定/キャンセル/全て）
- 期間
- プレイヤー（セレクト）
- 支払い方法

**一覧項目**
- 開始日時 / プレイヤー / 顧客名 / コース / 合計 / ステータス / 操作

**操作ボタン**
- 仮予約 → 「確定する」ボタン
- 確定 → 「キャンセル」「編集」
- キャンセル → 「復元」

### /owner/reservations/[id]（予約詳細）

- 全情報表示
- 金額内訳表示（コース / 指名料 / 延長 / オプション / 交通費 / 合計 / バック内訳）
- 確定 / キャンセル / 編集 / 削除 ボタン
- 変更履歴（簡易）

### /owner/players（プレイヤー管理）

**一覧**
- 名前 / ログインID / 状態 / 今月売上 / 操作（編集・無効化）

### /owner/players/[id]（プレイヤー編集）

**基本情報タブ**
- 名前 / ログインID / パスワードリセット / 有効無効
- 交通費バック率

**料金設定タブ**（ここが重要）
- コース一覧：各コースに対して「デフォルト適用」or「カスタム」選択
  - カスタム選択時：料金 / バック率を入力
- 指名料：初指名 / 本指名それぞれ同様
- 延長料金：延長種別ごと同様
- オプション：オプションごと同様

**設定がデフォルトか上書きかが一目で分かるUI**（バッジ表示）

### /owner/courses（コース編集）

- コース一覧テーブル
- 追加 / 編集 / 削除 / 並び替え
- 編集項目：名前 / 時間（分）/ 料金 / バック率 / 有効期間（イベント時）/ 有効無効

### /owner/nomination（指名料編集）

- 初指名・本指名の2行固定
- 編集：料金 / バック率

### /owner/extensions（延長料金編集）

- コースと同様の一覧UI

### /owner/options（オプション編集）

- オプション一覧
- 追加・編集・削除

### /owner/settings（店舗設定）

- 店舗名 / 連絡先
- 交通費入力方式（manual / fixed_by_area）
- オーナーパスワード変更

## スーパー管理者画面詳細

### /admin（全体ダッシュボード）

- 登録店舗数 / アクティブプレイヤー数 / 今月予約数
- 店舗別サマリ（売上・予約数）

### /admin/stores（店舗一覧）

- 一覧：店舗名 / store_code / オーナー名 / 状態 / 登録日
- 操作：編集 / 凍結 / なりすましログイン

### /admin/stores/new（店舗追加）

- 店舗名 / store_code / オーナー情報（名前・ログインID・初期パスワード）
- 初期マスタのテンプレート選択（空 / サンプル投入 / 他店舗コピー）

### /admin/stores/[id]（店舗詳細）

- 店舗情報編集
- オーナーアカウント再発行
- 全マスタに直接アクセス可能
- 全予約閲覧可能
- 凍結 / 削除

### なりすましログイン

- ボタン押下 → そのオーナーとしてログインしたセッションで別タブ起動
- `/owner` 以下が通常通り使える
- 戻る時は明示的にログアウトで管理者画面へ

---

## 共通UI要件

- **スマホファースト**：プレイヤー画面は特に徹底
- **金額表示**：3桁カンマ区切り、円マーク
- **日時**：YYYY/MM/DD HH:mm 形式
- **色設計**：
  - 仮予約：黄色系
  - 確定：緑系
  - キャンセル：グレー
- **ローディング表示**：全非同期処理
- **エラー表示**：トースト通知

## 画面遷移フロー（予約1件のライフサイクル）

```
[プレイヤー] /player/new
     ↓ 送信
[システム] 仮予約レコード作成（status=tentative）
     ↓
[プレイヤー] /player で仮予約として表示
[オーナー]  /owner/reservations で未確定として表示
     ↓ オーナーが確定ボタン押下
[システム] status=confirmed、金額・バック額を確定計算して保存
     ↓
[オーナー]  ダッシュボード売上に反映
[プレイヤー] /player/sales に反映
     ↓ （必要に応じてキャンセル）
[オーナー]  キャンセルボタン押下
[システム] status=cancelled、売上から除外
```
