## 技術選定メモ

| 項目 | 採用 | 理由 |
|---|---|---|
| フレームワーク | Next.js 15 App Router | Server ActionsでAPI実装が楽、Vercelと相性 |
| 言語 | TypeScript | 型安全、Supabase型自動生成と相性 |
| DB / Auth | Supabase | RLS、型生成、コスト面で最適 |
| UIライブラリ | Tailwind + shadcn/ui | カスタマイズ性と速度のバランス |
| フォーム | react-hook-form + zod | 複雑な予約入力のバリデーションに強い |
| グラフ | recharts | 軽量、Phase 2で十分 |
| ホスティング | Vercel | Next.jsと統合、無料枠で開始可 |
| パスワードハッシュ | bcrypt | 標準的 |

## Claude Codeへの投げ方推奨

タスク1個ずつ独立して投げる。例：

> 「Step 2-1 のマイグレーションファイルを `/supabase/migrations/001_initial_schema.sql` として作成して。仕様は `/docs/01_db_schema.md` を参照」

> 「Step 4-4 の予約作成Server Actionを `/app/player/new/actions.ts` に実装して。バリデーションはzod、スナップショット作成ロジック含む」

粒度が細かすぎると感じたら複数まとめて投げてもいい。ただし Step またぎはやめた方がいい（依存が絡んで失敗しやすい）
